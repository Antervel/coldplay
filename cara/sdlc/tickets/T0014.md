## T0014 — Add catch-all `handle_info` clauses to ChatLive and TeacherLive
git branch: `ai-agent/t0014_catch_all_handle_info`

**Problem:**
Per RULE 18, GenServers (and LiveViews, which are special GenServers) receive messages from monitors, timers, PubSub, and other unexpected sources beyond the ones you wrote handlers for. Without a catch-all fallback clause, an unexpected message crashes the LiveView process.

**Violation 1 — `ChatLive` (`lib/cara_web/live/chat_live.ex`):**

Has 14 `handle_info` clauses (lines 170-288) handling specific messages like `{:submit_message, ...}`, `{:llm_chunk, ...}`, `{:llm_end, ...}`, PubSub events, etc. But **no catch-all clause**. Any unexpected message — a malformed `ChatOrchestrator` reply, a stray monitor `{:DOWN, ...}` message, a PubSub message with an unexpected format — causes a `FunctionClauseError` that crashes the LiveView.

The relevant lines are:
- Line 170: `handle_info({:submit_message, message}, socket)`
- Line 175: `handle_info({:teacher_joined, _}, socket)`
- Line 188: `handle_info({:llm_chunk, branch_id, chunk}, socket)`
- Line 232: `handle_info({:llm_end, branch_id, full_text}, socket)`
- Line 256: `handle_info({:llm_status, branch_id, status}, socket)`
- Line 262: `handle_info({:update_tool_usage_counts, tool_usage_counts}, socket)`
- Line 268: `handle_info({:chat_started, _}, socket)`
- Line 270: `handle_info({:chat_state, _}, socket)`
- Line 272: `handle_info({:new_message, _}, socket)`
- Line 274: `handle_info({:message_deleted, _}, socket)`
- Line 276: `handle_info({:chat_left, _}, socket)`
- Line 280: `handle_info({:llm_metadata, _branch_id, _metadata}, socket)`
- Line 284: `handle_info({:llm_tool_called, _branch_id, _info}, socket)`
- Line 288: `handle_info({:llm_error, branch_id, error_message}, socket)`

**Violation 2 — `TeacherLive` (`lib/cara_web/live/teacher_live.ex`):**

Has 6 `handle_info` clauses (lines 25, 47, 58, 81, 101, 118) but **no catch-all clause**. Same crash risk.

**Solution:**

Add a catch-all `handle_info` clause at the **end** (after all specific clauses) in both LiveViews:

```elixir
@impl true
def handle_info(_message, socket) do
  # Unexpected message — log and ignore rather than crashing the LiveView
  {:noreply, socket}
end
```

The catch-all should come **after** all specific clauses because Elixir pattern-matches top-to-bottom.

**Important placement:** In `ChatLive`, the `handle_info` callbacks span lines 170-288. The catch-all must be added **below line 288** (or wherever the last specific handler is). In `TeacherLive`, it goes below line 118.

Optionally, add a `Logger.warning` call in the catch-all to make unexpected messages observable without crashing:

```elixir
@impl true
def handle_info(message, socket) do
  Logger.warning("ChatLive received unexpected message: #{inspect(message)}")
  {:noreply, socket}
end
```
