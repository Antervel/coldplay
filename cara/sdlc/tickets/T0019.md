## T0019 — Add observability (telemetry/logging) to `Cara.Audit` context
git branch: `ai-agent/t0019_audit_observability`

**Problem:**
Per RULE 39, critical paths should emit useful telemetry or logs to make failures observable. The `Cara.Audit` context — which handles persistence of student chat sessions and messages — has **zero observability**:

- `create_session/1` (audit.ex:21-25) — no log entry on success or failure
- `get_session/1` (audit.ex:31-33) — no log entry
- `list_branches/1` (audit.ex:58-115) — no log entry for query timing or results count
- `list_messages_for_branch/2` (audit.ex:130-137) — no log entry

When audit data goes missing (e.g., a student's messages don't appear in the logs view), there is zero way to diagnose whether:
- The session was never created
- The messages were never persisted
- The query is failing
- The database connection is down

Compare with other modules in the project that do have observability:
- `Cara.ContentClassifier` logs timing and results
- `Cara.AI.Chat` has OpenTelemetry spans
- `Cara.AI.Tools.*` log execution times
- `Cara.Wikipedia` logs API call timing

**Solution:**

Add `Logger.info` / `Logger.warning` calls (with **scrubbed data** — see T0007 for PII guidance) to the key Audit functions. Use structured logging where possible.

**For `create_session/1`:**
```elixir
def create_session(attrs) do
  result = %Session{}
    |> Session.changeset(attrs)
    |> Repo.insert()
  
  case result do
    {:ok, session} ->
      Logger.info("Audit session created chat_id=#{session.chat_id}")
    {:error, changeset} ->
      Logger.error("Audit session creation failed: #{inspect(changeset.errors)}")
  end
  
  result
end
```

**For `list_branches/1`:**
```elixir
def list_branches(opts \\ []) do
  start_time = :erlang.monotonic_time(:millisecond)
  result = ... # existing query
  
  end_time = :erlang.monotonic_time(:millisecond)
  {branches, total_count} = result
  Logger.info("Audit list_branches returned #{total_count} total, #{length(branches)} in page, took #{end_time - start_time}ms")
  
  result
end
```

**For `list_messages_for_branch/2`:**
```elixir
def list_messages_for_branch(chat_id, branch_id) do
  start_time = :erlang.monotonic_time(:millisecond)
  result = ... # existing query
  
  end_time = :erlang.monotonic_time(:millisecond)
  Logger.info("Audit list_messages_for_branch(chat_id=#{chat_id}, branch_id=#{branch_id}) returned #{length(result)} messages, took #{end_time - start_time}ms")
  
  result
end
```

**Important:** Do NOT log the actual message content or full structs — only log IDs, counts, and timings to avoid PII leakage (see RULE 30).
