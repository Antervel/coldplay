## T0006 — Migrate `TeacherLive` chats to LiveView streams for memory efficiency
git branch: `ai-agent/t0006_teacher_live_streams

**Problem:**
Per DMV Prompt Rule 2, collections that can exceed ~100 items should use LiveView streams (`stream/3`, `stream/4`) instead of regular assigns to avoid O(n) memory per connected user. The `TeacherLive` dashboard stores active student chats in a regular `@chats` map assign.

As the number of concurrent student sessions grows:
- Full chat state for every student is duplicated in memory for every connected teacher
- No streaming-based diffing (full state sent on every update)
- `PHX_SERVER=true` deployments are especially sensitive since there is no client-side state

**Solution:**
Replace the regular `assign(socket, :chats, %{})` with LiveView streams. Each student chat becomes a stream item. The teacher dashboard template is updated to use the `phx-update="stream"` pattern.

Key design decisions:
- Each chat stream item uses a unique stable ID (the student's `chat_id`)
- `handle_info` callbacks use `stream_insert/4` for insertions and updates
- `stream_delete/3` for removals
- Empty state is handled via a tracking assign (`@chats_empty?`) since streams don't support empty-state queries
- The `chats` assign currently tracks all chats in a map — replaced by stream internals
