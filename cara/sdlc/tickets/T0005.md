## T0005 — Default `ContentClassifier` and `Wikipedia` to `Cara.HTTPClient` instead of raw `Req`
git branch: `ai-agent/t0005_http_client_wrapper

**Problem:**
Per DMV Prompt Rule 20, third-party library APIs should always be wrapped behind a project-owned module. `Cara.HTTPClient` already exists and wraps `Req`, but `ContentClassifier` and `Wikipedia` both have their own `http_client()` helper that defaults directly to the `Req` module.

This creates two issues:
1. Direct coupling: business-logic modules are directly coupled to `Req` instead of the project's own `Cara.HTTPClient` abstraction
2. Harder to mock: `Mox`-based testing via `Cara.HTTPClient` is not possible since these modules bypass it

**Solution:**
Change the default value in both `http_client()` helpers from `Req` to `Cara.HTTPClient`. Both functions already accept a configurable client via `Application.get_env(:cara, :http_client, ...)` — no structural changes needed.

However, `Cara.HTTPClient.get/2` uses `Req.new(opts) |> OpentelemetryReq.attach(...) |> Req.get(url: url)` while `ContentClassifier` and `Wikipedia` use `http_client().post(...)` and `http_client().get(..., params: ..., headers: ...)`. `Cara.HTTPClient` only exposes `get/2` not `post/2`. So we need to:
1. Add `post/3` to `Cara.HTTPClient` (and its `@callback` if defined)
2. Change the defaults in both modules
3. Update tests if they mock at the `Req` level
