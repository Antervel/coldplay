## T0012 — Server-side re-verification of client-supplied IDs in chat event handlers
git branch: `ai-agent/t0012_verify_client_ids_in_handle_event`

**Problem:**
Per RULE 15, event params come from the client (WebSocket) and can be edited before they reach the server. Several `handle_event` callbacks in `ChatLive` pass client-supplied IDs directly into business logic without verifying they belong to the current session.

**Violations in `lib/cara_web/live/chat_live.ex`:**

1. **Line 96** — `handle_event("switch_branch", %{"id" => id}, socket)` — client supplies a branch ID, passed to `ChatService.switch_branch(socket.assigns.branched_chat, id, socket)`

2. **Line 106** — `handle_event("branch_off", %{"id" => id}, socket)` — client supplies a message ID, passed to `ChatService.branch_off(socket.assigns.branched_chat, id, socket)`

3. **Line 146** — `handle_event("delete_message", %{"id" => id}, socket)` — client supplies a message ID, passed to `ChatService.delete_message(socket.assigns.branched_chat, id, socket)`

**Risk analysis:** The risk is partially mitigated because the `BranchedChat` struct is in-memory per-socket-assigns — so a malicious client can only affect their own session's branches. However, a malformed or non-existent `id` could:
- Cause a pattern-match crash in `ChatService` (returns `{:noreply, socket}` but the error is swallowed)
- Allow switching to a non-existent branch (which may cause confusing UI state)
- Allow "branching off" or deleting from a non-existent message (which may corrupt the branch structure)

**Solution:**

Add guards in each handler that verify the client-supplied ID exists in the current `branched_chat` before proceeding.

**For `switch_branch`:** Check the branch ID exists in `branched_chat.branches`:
```elixir
def handle_event("switch_branch", %{"id" => id}, socket) do
  if Map.has_key?(socket.assigns.branched_chat.branches, id) do
    branched_chat = ChatService.switch_branch(socket.assigns.branched_chat, id, socket)
    {:noreply, socket |> assign_branched_chat(branched_chat) |> push_event("rendered", %{})}
  else
    {:noreply, socket}
  end
end
```

**For `branch_off` and `delete_message`:** Extract a helper that checks whether the message ID exists in any branch of the current `branched_chat`:
```elixir
defp message_exists_in_chat?(branched_chat, message_id) do
  Enum.any?(branched_chat.branches, fn {_branch_id, branch} ->
    Enum.any?(branch.messages, &(&1.id == message_id))
  end)
end
```

Then guard each handler:
```elixir
def handle_event("branch_off", %{"id" => id}, socket) do
  if message_exists_in_chat?(socket.assigns.branched_chat, id) do
    branched_chat = ChatService.branch_off(socket.assigns.branched_chat, id, socket)
    {:noreply, socket |> assign_branched_chat(branched_chat) |> push_event("rendered", %{})}
  else
    {:noreply, socket}
  end
end
```
