## T0007 — Scrub PII from log statements in `Chat` and `ContentClassifier`
git branch: `ai-agent/t0007_scrub_pii_from_logs`

**Problem:**
Per RULE 30, two `Logger.info` calls dump full structs that contain Personally Identifiable Information (PII) — student conversation text — into logs. This is a compliance risk.

**Violation 1 — `lib/cara/ai/chat.ex:153`:**
```elixir
Logger.info("LLM call_llm starting with context: #{inspect(context)}")
```
`context` is a `ReqLLM.Context` struct. It contains the **entire conversation history** — every user and assistant message in full text. A student might type their name, address, or other PII into the chat. This call fires on **every** LLM call, so every message a student sends gets logged in full.

**Violation 2 — `lib/cara/content_classifier.ex:113`:**
```elixir
Logger.info("safe_result?: classified as #{inspect(result)}")
```
The `result` map contains the original `input` field (the user's text) alongside classification scores like `%{"input" => "Hi my name is Johnny and I live at...", "sexual" => %{"score" => 0.001}, ...}`. This logs whatever the student typed verbatim.

**Solution:**

Replace each PII-leaking log line with a scrubbed equivalent:

**For `lib/cara/ai/chat.ex:153`:**
Replace:
```elixir
Logger.info("LLM call_llm starting with context: #{inspect(context)}")
```
With:
```elixir
Logger.info("LLM call_llm starting with context: #{context_messages_count(context.messages)} messages, #{context_total_chars(context.messages)} chars")
```
Add helper functions:
```elixir
defp context_messages_count(messages), do: length(messages)
defp context_total_chars(messages) do
  Enum.reduce(messages, 0, fn %{content: content}, acc -> acc + String.length(content || "") end)
end
```
Or more simply, just log the message count and total character count without exposing any content.

**For `lib/cara/content_classifier.ex:113`:**
Replace:
```elixir
Logger.info("safe_result?: classified as #{inspect(result)}")
```
With something that logs only the scores and thresholds without the `input` field:
```elixir
scores_only = Map.drop(result, ["input"])
Logger.info("safe_result?: classified as #{inspect(scores_only)}")
```
This drops the `"input"` key (the user's original text) before logging, keeping only the classification scores (toxicity, sexual, etc.), which are non-PII numerical values.
