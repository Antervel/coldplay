## T0002 — Remove warning message "use of unverified model"
git branch: `ai-agent/t0002_remove_warning_message_use_unverified_model

**Problem:**
When running this software, when the user sends a message, the software sends a req_llm request using a custom module. There is a warning in the console that pollutes it and makes debugging harder:
```
warning: Using unverified model: openai:cara-cpu
...
```

**Solution:**
The warning occurs because `ReqLLM.stream_text/3` receives a raw model string (`"openai:cara-cpu"`) which triggers a fallback path in `ReqLLM.model/1` when the model isn't found in the LLMDB catalog.

Fix: Add a `resolve_model/1` function in `Cara.AI.Chat` that converts the `"provider:model_id"` string into an inline model spec map (`%{provider: :openai, id: "cara-cpu"}`) via `ReqLLM.model!/1`. This inline spec path skips the LLMDB lookup entirely, suppressing the warning.

Changes:
- `lib/cara/ai/chat.ex`:
  - Added `resolve_model/1` to parse model string and produce an inline model struct
  - Called `resolve_model` in `build_config/1` so both `send_message` and `send_message_stream` use the resolved model
  - Updated `call_llm` type spec from `String.t()` to `ReqLLM.model_input()`
  - Fixed OpenTelemetry span attribute and log line to use `model_alias` (string) instead of the struct
