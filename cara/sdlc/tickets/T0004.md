## T0004 — Implement toggling models in the /settings page
git branch: `ai-agent/t0004_-_Implement_toggling_models_settings

**Problem:**
We need a way of easily switching between between models. The current model is cara:cpu in Ollama. I would like to be able to have a select in the `/settings` page that would reconfigure the model to be used.
The next model will be:
```
base_url = "https://integrate.api.nvidia.com/v1",
model="openai/gpt-oss-20b",
api_key: $NVIDIA_API_KEY
```

This is a tricky ticket. At the very least we have to:
* Add a select in /settings ( please force the background to white and the font to black )
* Reconfigure branched_llm to change switch models. This will be another ticket so don't do anything, but prepare a document with the steps to do this, so it can be done in the branched_llm repo.


**Solution:**
The settings page at `/settings` is updated to include a model selector (HTML `<select>`) with two options:
- `openai:cara-cpu` (Ollama, local)
- `openai:gpt-oss-20b` (NVIDIA, cloud, requires `$NVIDIA_API_KEY`)

On form submission, the selected model string is persisted via `Application.put_env(:cara, :ai_model, ...)` and used immediately by `Cara.AI.Chat` (which reads `:cara, :ai_model` at call time via `default_model/0`).

A supporting document (`docs/RECONFIGURE_BRANCHED_LLM.md`) is provided describing the steps needed in the `branched_llm` repo to fully support switching LLM providers (base URL + API key per provider), since the current implementation uses a single global `:req_llm` config.
