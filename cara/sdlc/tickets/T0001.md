## T0001 — Replace Wikipedia API search with custom search API
git branch: `ai-agent/t0001_replace_wikipedia_search_with_custom_api`

**Problem:**
The `wikipedia_search` tool currently queries the Wikipedia API directly via `https://en.wikipedia.org/w/api.php`. We want to replace this with a custom-made search API that provides semantic/full-text search results.

**Solution:**
Replace the call in `Cara.Wikipedia.search_articles/1` to query a custom API endpoint instead of the Wikipedia API. The custom API has the following interface:

```
curl "http://localhost:8001/search?q=What+is+Photosynthesis&k=5"
{"query":"What is Photosynthesis","results":[{"rank":1,"title":"Photosynthesis\n","score":0.903},{"rank":2,"title":"Photosymthesis\n","score":0.8278},{"rank":3,"title":"Photosynthesise\n","score":0.8224},{"rank":4,"title":"Photosynthesising\n","score":0.816},{"rank":5,"title":"Photosynthesis_Research\n","score":0.7719}],"elapsed_ms":34.6}
```

The base URL and parameters (`q` for query, `k` for number of results) should be configurable via Application env, defaulting to `http://localhost:8001/search`. The response parsing should map the custom API's result format to the same struct shape expected by callers (`%{title: ..., extract: ..., url: ...}`), adapting fields as needed (e.g., empty strings for `extract`; `url` is "https://en.wikipedia.org/wiki/#{title}" if not provided by the custom API).
