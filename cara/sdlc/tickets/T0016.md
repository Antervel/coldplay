## T0016 â€” Handle return values of `Repo.insert` in fire-and-forget paths
git branch: `ai-agent/t0016_handle_repo_insert_results`

**Problem:**
Per RULE 25, two locations silently discard the return value of `Repo.insert()`, making failures invisible.

**Violation 1 â€” `lib/cara/plugins/audit_plugin.ex:39-44`:**

```elixir
defp default_insert(attrs) do
  Task.Supervisor.start_child(Cara.TaskSupervisor, fn ->
    %Message{}
    |> Message.changeset(attrs)
    |> Repo.insert()
  end)
end
```

The `Repo.insert()` call runs inside a fire-and-forget Task. The return value (`{:ok, %Message{}}` or `{:error, changeset}`) is never inspected. If the insert fails (unique constraint violation, database connection error, changeset validation error), the error is silently swallowed. The developer only discovers the issue when audit logs are missing.

**Violation 2 â€” `lib/cara/ai/chat.ex:230-233`:**

```elixir
{:ok, result} = success ->
  ToolCache.save_result(tool.name, args, result)  # return ignored
  success
```

The return value of `ToolCache.save_result/3` (which calls `Repo.insert()`) is discarded. If the tool result can't be saved, the cache miss will cause the tool to re-execute on the next identical call â€” but the calling code proceeds as if caching succeeded.

**Solution:**

**Fix 1 (`audit_plugin.ex`):** Log the error when the insert fails inside the Task:

```elixir
defp default_insert(attrs) do
  Task.Supervisor.start_child(Cara.TaskSupervisor, fn ->
    %Message{}
    |> Message.changeset(attrs)
    |> Repo.insert()
    |> case do
      {:ok, _message} -> :ok
      {:error, changeset} ->
        Logger.error("Failed to persist audit message: #{inspect(changeset.errors)}")
    end
  end)
end
```

Also add `require Logger` at the top of the module (if not already present).

**Fix 2 (`chat.ex`):** Log the error when `save_result` fails:

```elixir
{:ok, result} = success ->
  case ToolCache.save_result(tool.name, args, result) do
    {:ok, _} -> :ok
    {:error, _changeset} ->
      Logger.warning("Failed to cache tool result for #{tool.name}")
  end
  success
```
