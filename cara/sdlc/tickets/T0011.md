## T0011 — Add PubSub unsubscribe calls to match subscriptions in mount
git branch: `ai-agent/t0011_pubsub_unsubscribe_on_terminate`

**Problem:**
Per RULE 3, two LiveViews subscribe to the `"teacher:monitor"` PubSub topic in `mount/3` but never unsubscribe. Over multiple reconnects (e.g., due to LiveView crashes, network issues, or the user navigating away and back), orphan subscriptions accumulate on the PubSub side.

**Violation 1 — `lib/cara_web/live/teacher_live.ex:10`:**
```elixir
if connected?(socket) do
  if monitoring_enabled do
    Phoenix.PubSub.subscribe(Cara.PubSub, "teacher:monitor")
```
No `terminate/2` callback exists in `TeacherLive` at all, so there is no unsubscribe.

**Violation 2 — `lib/cara/education/session.ex:94-96`** (called by `ChatLive`):
```elixir
defp do_subscribe_and_broadcast(%__MODULE__{} = session, socket) do
  if connected?(socket) and Monitoring.monitoring_enabled?() do
    Monitoring.subscribe()
    Monitoring.broadcast_chat_started(socket, session.student_info)
  end
```
`ChatLive` has a `terminate/2` (`chat_live.ex:62-65`) but it only broadcasts `chat_left` — it never calls `Phoenix.PubSub.unsubscribe`.

**Where `Monitoring.subscribe()` lives** — `lib/cara/education/monitoring.ex:38-40`:
```elixir
def subscribe do
  Phoenix.PubSub.subscribe(Cara.PubSub, "teacher:monitor")
end
```

**Solution:**

Add `terminate/2` callbacks to both `TeacherLive` and `ChatLive` that call `Phoenix.PubSub.unsubscribe(Cara.PubSub, "teacher:monitor")`.

**For `TeacherLive` (`lib/cara_web/live/teacher_live.ex`):**
Add a `terminate/2` callback:
```elixir
@impl true
def terminate(_reason, socket) do
  if socket.assigns[:monitoring_enabled] do
    Phoenix.PubSub.unsubscribe(Cara.PubSub, "teacher:monitor")
  end
  :ok
end
```

**For `ChatLive` (`lib/cara_web/live/chat_live.ex`):**
Update the existing `terminate/2` at line 62 to also unsubscribe:
```elixir
@impl true
def terminate(_reason, socket) do
  if Map.has_key?(socket.assigns, :chat_id) do
    Session.broadcast_left(socket, socket.assigns.chat_id)
  end
  if Monitoring.monitoring_enabled?() do
    Monitoring.unsubscribe()
  end
  :ok
end
```

Also add an `unsubscribe/0` function to `Monitoring` module:
```elixir
# in lib/cara/education/monitoring.ex
def unsubscribe do
  Phoenix.PubSub.unsubscribe(Cara.PubSub, "teacher:monitor")
end
```
